《Devops》 - devops背景

2021-07-30

滚动更新

系统由多个组件组成，在不停止服务的前提下，每次只更新部分组件

开发团队和运维团队如何共享信息？

常存在，开发团队，上线新功能，访问量陡增 或者出现大量异常，运维不知道，导致长时间服务不可用状态

  • 因此开发和运维，需要了解工作内容，一起参与
  • 敏捷开发 → 持续的运维问题 → 配置管理工具 → devops产生 → 配置工具和沟通工具结合，提升服务质量

康威定律

大意为： 系统架构,等同于组织沟通结构

devops

  • 专业性越强团队，信息沟通，就会带来损耗，沟通越多，损耗就越大，
  • 此时引入devops，和支撑devops的各种工具，是系统架构、组织结构同时发生变化，将会为组织带来很多积极影响
  • devops 不仅需要工具，还需要组织文化变动，实践devops可以消除对个人的依赖、减少团队损耗、提高产品质量。
