性能优化从js规范开始

1.命名通俗易懂，知名达意

2.结构不要过深

拆分代码，避免过多深入调用，函数结构化，相关代码存放一起，把逻辑和该逻辑处理的数据放在相近的位置，往往同时修改，不会遗漏

3.扩展性强

代码随时可拓展，且 不影响相关调用代码，封装成接口便于调用

4.删除无用code

5.代码集中、模块化管理
